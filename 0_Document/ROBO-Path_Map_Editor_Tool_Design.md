# ROBO-Path 맵 에디터 도구 설계 (Map Editor Tool Design)

## 1. 목적 및 배경
- 블록마다 좌표를 수동 입력하는 현재 방식은 대규모 맵 제작에 불가능하다(10×10 블록으로 500m 한 변에 50개, 바닥만 2,500개, 높이 적층 포함 시 수천~수만 블록).
- 마인크래프트식으로 격자에 블록을 클릭 배치/적층하는 Unity 에디터 확장 도구가 필요하다.
- 구현 방식은 런타임 게임이 아니라 **Unity 에디터 확장 툴**(Editor 스크립트)로 한다. 목표는 맵 제작 효율화이지 맵 에디터 게임 제작이 아니다.

## 2. 블록 규격 (확정된 설계 반영)
- 모든 블록은 footprint(바닥 면적) 10m × 10m 고정. (인접 판정 일관성 보장)
- 블록 높이는 2/4/6/8/10의 5단계로 존재한다.
- 블록 종류:
  - 평지 블록(= 기본 블록): 윗면이 수평. 높이 5종.
  - 경사 블록: 윗면이 한 방향으로 비스듬히 상승하는 쐐기. 상승 높이 5종(2/4/6/8/10).
  - (향후) 계단 블록 등은 동일 규격 체계로 확장.
- 적층 규칙:
  - 블록을 격자 칸 위에 쌓아 올려 지형 높이를 만든다.
  - 가장 위에 쌓인 블록의 윗면이 지면(로봇 주행면)이 된다.
  - 높이가 낮은 블록 위에 높은 블록을 쌓는 것도 허용한다. (최소 블록 사용 관점에서는 비효율적일 수 있으나, 향후 기둥 등 오브젝트 적층 호환성을 위해 규칙으로 허용)
  - 높이별 평지 블록이 존재하는 이유: 경사로 높아진 평지가 넓게 이어질 때 높이 2 블록만 쌓으면 블록 수가 과다해지므로, 높이 10 블록 하나로 처리할 수 있어야 한다.

## 3. 좌표/격자 체계
- XZ 평면을 10m 단위 격자로 나눈다. 블록은 격자 칸에 스냅되어 배치되므로 좌표 수동 계산이 불필요하다.
- Y(높이)는 적층 단위로 관리한다. 블록을 쌓으면 아래 블록 윗면 높이에 맞춰 자동 적층된다.
- 격자 인덱스(예: gx, gz)와 적층 높이로 블록 위치를 표현하여, 내부적으로 월드 좌표를 자동 산출한다.

## 4. 기능 명세
### 4.1 최소 필수 기능
- 격자 위에서 클릭으로 블록 배치 / 삭제.
- 블록 종류 및 높이 선택 (평지/경사 × 높이 2~10).
- 경사 블록의 상승 방향 회전(동서남북 4방향).
- 적층: 기존 블록 위에 얹기(윗면 높이 자동 정렬).
- 격자 스냅 자동(좌표 수동 입력 불필요).
- 배치 시 블록 종류에 따라 적절한 태그 자동 부여 (Terrain_Flat, Terrain_Slope, Path_Ramp 등 맵 설계 명세의 태그 체계와 연계).

### 4.2 후순위 기능
- 맵 데이터 저장/불러오기(격자 인덱스+높이+종류+방향의 직렬화 포맷). Unity 씬과 별개로 맵 구성을 데이터로 보관하여 재현 가능하게 한다.
- 블록 팔레트 UI(종류/높이 선택 패널).
- 영역 단위 일괄 배치(드래그로 평지 채우기 등).

### 4.3 향후 연계 (명시만)
- 씬 덤프 도구와의 연계: 배치 결과가 씬 덤프(블록/인접)로 정확히 추출되어야 한다.
- 거점 타일(Node_Station 등) 배치 기능.
- 블록 프리팹화: 검증된 블록 5종(+경사)을 프리팹으로 정규화한 뒤, 본 도구가 해당 프리팹을 인스턴스화하여 배치한다.

## 5. 블록 프리팹 의존성 (중요)
- 본 도구는 "검증된 블록 프리팹"을 재료로 사용한다.
- 따라서 도구 구현에 앞서, 경사/평지 블록의 규격이 NavMesh 연결 및 씬 덤프 인접 판정 측면에서 검증되어야 한다.
- 구현 순서: (1) 블록 규격 수동 검증 → (2) 검증된 블록 프리팹화 → (3) 본 맵 에디터 도구 구현. 본 문서는 (3)의 설계이며, (1)이 완료되기 전에는 구현하지 않는다.

## 6. 구현 범위 및 비범위
- 범위: Unity 에디터 내 격자 기반 블록 배치/적층/삭제, 태그 자동 부여, 맵 데이터 저장/불러오기.
- 비범위: 런타임 인게임 에디터, 자유 형상 모델링, 물리 시뮬레이션. (블록은 정규 규격 프리팹의 조합으로만 구성)
